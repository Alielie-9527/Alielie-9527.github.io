LLM微调之LoRA篇（二）

2026/1/8 LLM学习 LoRA 大模型微调

QLoRA 就是在 LoRA 的基础上引入了量化。大模型训练过程引入了量化，原因还是为了进一步减少显存占用（关于显存占用也会整理相关笔记）。因为模型参数本身需要加载到显存中才能进行计算，比如一个 7B 模型推理就需要 14~16 GB（采用 FP16 或 BF16），即使是 LoRA 微调也需要 20 多个 G 的显存（还要考虑输入数据的 batch_size 等参数）。

QLoRA 的原理

QLoRA 有以下三个组成部分：

一、 4-bit NormalFloat 量化

​ 为什么以前的量化方式不好？ 以 4-bit 量化为例，实际上量化过程就是将参数映射到 [−7,7][-7, 7][−7,7] 之间的 15 个数（刻度），但是大模型参数的分布往往是不均匀的，而是呈现近似正态分布，大部分值都在零附近，而量化过程会导致这些数值之间的差异被抹平。而值域两侧的数值很少用到，浪费了存储空间。

​ 引入分位数量化，保证每一个量化刻度里包含的数据个数是相等的。比如量化成 4 和量化成 5 的数的个数是相等的（理论上），这是信息上最优数据类型。这样输入权重理论上均匀分布在所有刻度上，信息利用率是最高的。

补充一点前置知识：分位数函数以及分布函数（CDF，又叫累积分布函数）

1. 分布函数 (CDF)

定义：设 XXX 是一个随机变量，其分布函数 (Cumulative Distribution Function) 记为 F(x)F(x)F(x)。

公式：

含义：F(x)F(x)F(x) 表示随机变量 XXX 取值小于等于 xxx 的概率。

​ NF4 维护一个 NF4 表，表中维护的就是 16 个数，权重量化成了这 16 个数在表的序号（16 个序号只需要用 4-bit 表示）。

3. 数学构造：构造 NF4 的 16 个数

​ 首先必须要有一个精确的 0，也就是权重中 0 要映射到 0。因为大部分的参数是集中在 0 附近的，直接量化成 0 引起的误差是最小的，同时需要 0 来切断一些信号往更深层传递。

​ 剩下还有 15 个点分别位于正半轴和负半轴。由于 15 是奇数，只能不对称分配：7 个负数，8 个正数。

​ 论文中理想的计算公式如下（其中 Q(⋅)Q(·)Q(⋅) 为标准正态分布的累积分布函数的反函数）：

qi=12(Q ⁣(i17)+Q ⁣(i+117)),i=0,…,15 q_i = \frac{1}{2}\left(Q\!\left(\frac{i}{17}\right) + Q\!\left(\frac{i+1}{17}\right)\right), \quad i = 0, \ldots, 15 qi​=21​(Q(17i​)+Q(17i+1​)),i=0,…,15

A. 计算左边负数（基于概率划分）

  • 现状：有 7 个负数。
  • 假设：为了让这 7 个负数合理分布，假设整体是 15 等分（7 左 + 1 中 + 7 右）。
  • 计算如下：
    • 15 等分，区间划分为 15 个（如图 1 所示，划分为 15 个面积相等的区间），切分点对应的 F(x)F(x)F(x) 分别为：0,1/15,2/15,…,15/150, 1/15, 2/15, \dots, 15/150,1/15,2/15,…,15/15。

B. 计算右侧

​ 右侧计算同上，只是右侧最右边对应边界概率为 1−132=31321 - \frac{1}{32} = \frac{31}{32}1−321​=3231​。

实际操作上取平均：offset=(P15+P16)/2≈0.9677083\text{offset} = (P_{15} + P_{16}) / 2 \approx 0.9677083offset=(P15​+P16​)/2≈0.9677083。这一步是为了归一化服务，使得最大值和最小值的绝对值相同，保证归一化后的表范围在 [−1,1][-1, 1][−1,1]。

结合代码来看具体操作：

def create_nf4_map():
    # 1. 核心参数：边界概率的妥协值
    offset = 0.9677083 

    # 2. 切分概率区间 (linspace)
    # 正半轴 (对应 8 个数)：从概率为 offset 开始等间隔取 9 个点到 0.5（对应 0 ），每个点对应的就是划分的区间中点
    # 生成 9 个点，[:-1] 去掉 0.5 (因为 0.5 对应 0，要单独处理)
    step_pos = torch.linspace(offset, 0.5, 9)[:-1]
    
    # 负半轴 (对应 7 个数)
    # 生成 8 个点，[:-1] 去掉 0.5
    step_neg = torch.linspace(offset, 0.5, 8)[:-1]

    # 3. 正态分布映射 (PPF: 概率 -> 数值)
    # norm.ppf 是累积分布函数的反函数
    v_pos = norm.ppf(step_pos).tolist()       # 正数部分
    v_neg = (-norm.ppf(step_neg)).tolist()    # 负数部分 (利用对称性计算)
    
    # 4. 拼接 (Stitch)
    # 7 个负数 + 1 个精确的零 + 8 个正数 = 16 个数
    v = v_pos + [0.0] + v_neg
    
    # 5. 归一化 (Normalize)
    values = torch.Tensor(v)
    values = values.sort().values
    # 除以最大值，将范围强制压缩到 [-1, 1]
    # 目的：制作一把通用的“标准尺子”
    values /= values.max()
    
    return values

M=max⁡j∣wj∣ M = \max_{j} |w_j| M=jmax​∣wj​∣
  • 存储：这个 MMM (FP32/BF16) 需要被额外存储下来，用于后续还原。

步骤二：归一化 (Normalization)，将块内所有权重除以 MMM，将其映射到 [−1,1][-1, 1][−1,1] 的标准范围内：

xj=wjM x_j = \frac{w_j}{M} xj​=Mwj​​
  • 此时，xjx_jxj​ 的分布完美契合 NF4 的设计范围。

二、双重量化

​ 模型存储时，量化常数也需要额外存储。双重量化的体现就是对量化参数本身再进行一次量化。QLoRA 中量化参数也就是上文中的 MMM。注意这里采取的是 FP8 量化，因为对量化参数的精度要求比较高，否则会进一步引入误差。

采用双重量化前显存计算：

  • 权重本身：每个参数 4-bit (0.5 Byte)。
  • 量化常数：每 64 个参数产生一个 FP32 的 ccc (4 Bytes)。
    • 平均每个参数的额外开销：4 Bytes/64=0.0625 Bytes4 \text{ Bytes} / 64 = 0.0625 \text{ Bytes}4 Bytes/64=0.0625 Bytes。
    • 也就是每个参数占 0.5 bits。

采用双重量化后：

  1. 权重：4-bit。
  2. 一级常数：8-bit (原来是 32-bit)。
  3. 二级常数：32-bit (但数量极少，每 64×25664 \times 25664×256 个权重才有一个)。

显存计算：

  • 一级常数开销：8 bits/64=0.125 bits/param8 \text{ bits} / 64 = 0.125 \text{ bits/param}8 bits/64=0.125 bits/param。
  • 二级常数开销：32 bits/(64×256)≈0.002 bits/param32 \text{ bits} / (64 \times 256) \approx 0.002 \text{ bits/param}32 bits/(64×256)≈0.002 bits/param。
  • 总额外开销：约 0.127 bits/param。

三、分页优化

​ 利用 NVIDIA 统一内存（Unified Memory）特性，在 GPU 偶尔内存不足的场景中，实现 CPU 与 GPU 之间的自动分页传输，以确保 GPU 处理过程不因 OOM（Out of Memory）而中断。

YBF16=XBF16doubleDequant(c1FP32,c2k-bit,WNF4)+XBF16L1BF16L2BF16 \mathbf{Y}^{\text{BF16}} = \mathbf{X}^{\text{BF16}} \text{doubleDequant}(c_{1}^{\text{FP32}}, c_{2}^{\text{k-bit}}, \mathbf{W}^{\text{NF4}}) + \mathbf{X}^{\text{BF16}} \mathbf{L}_{1}^{\text{BF16}} \mathbf{L}_{2}^{\text{BF16}} YBF16=XBF16doubleDequant(c1FP32​,c2k-bit​,WNF4)+XBF16L1BF16​L2BF16​

​ 其中，doubleDequant(⋅)\text{doubleDequant}(\cdot)doubleDequant(⋅) 表示双重量化的反量化过程，c1c_1c1​ 和 c2c_2c2​ 分别表示两级量化常数，WNF4\mathbf{W}^{\text{NF4}}WNF4 表示 NF4 量化后的权重矩阵。
